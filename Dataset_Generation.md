# Спецификация генерации датасета

Данный документ описывает процесс создания синтетического набора данных для обучения и валидации ML-моделей. Датасет генерируется на основе физической модели волн Капицы.

## 1. Параметрическое пространство (Sampling)
Для обеспечения репрезентативности данных и предотвращения смещения выборки (bias), используется два типа распределений при выборе параметров:

| Параметр | Обозначение | Диапазон значений | Распределение | Единицы измерения |
| :--- | :---: | :---: | :---: | :---: |
| **Динамическая вязкость** | $\mu$ | $[0.001, 1.0]$ | Log-uniform | Па·с |
| **Средняя толщина пленки** | $h_0$ | $[0.0002, 0.002]$ | Log-uniform | м |
| **Плотность жидкости** | $\rho$ | $[800, 1200]$ | Uniform | кг/м³ |
| **Поверхностное натяжение** | $\sigma$ | $[0.02, 0.08]$ | Uniform | Н/м |
| **Амплитуда возмущения** | $\delta$ | $\epsilon \cdot h_0$, где $\epsilon \in [0.001, 0.05]$ | Uniform ($\epsilon$) | м |

**Обоснование Log-uniform сэмплирования**: 
Для параметров $\mu$ и $h_0$, значения которых охватывают несколько порядков величины, обычное равномерное распределение привело бы к недопредставленности малых значений. Логарифмическое сэмплирование обеспечивает равную вероятность выбора значения из каждого порядка (например, интервал $[0.001, 0.01]$ будет представлен так же часто, как $[0.1, 1.0]$), что критически важно для обучения нейросети распознавать различные режимы течения.

**Обоснование относительного возмущения ($\delta$)**:
Чтобы избежать численной неустойчивости при очень тонких пленках и обеспечить репрезентативность данных, амплитуда начального возмущения $\delta$ задается как доля от текущей средней толщины $h_0$. Это гарантирует, что уровень шума всегда пропорционален масштабу системы.

**Примечание**: Высота стенки $L = 0.5$ м остается константой для всех сценариев.

## 2. Конфигурация датчиков (Sensor Setup)
Для обеспечения универсальности модели, схема расположения датчиков рандомизируется для каждого примера:
- **Количество датчиков ($k$)**: Случайное целое число в диапазоне от $1$ до $10$.
- **Позиции датчиков ($x_i$)**: Случайные точки в интервале $[0.05, 0.45]$ м. Это позволяет избежать краевых эффектов у входа и выхода.
- **Сортировка**: Позиции датчиков всегда сортируются по возрастанию $x_1 < x_2 < \dots < x_k$.

## 3. Временные параметры сбора данных
Для оптимизации времени генерации и обеспечения репрезентативности сигнала установлены следующие параметры:
- **Общее время симуляции**: $0.2 \dots 0.3$ с. Этого достаточно для выхода системы на установившийся режим волнового течения.
- **Окно наблюдения ($T_{window}$)**: Последние $0.1$ с симуляции используются для формирования временного ряда.
- **Дискретизация**: 
    - Количество измерений в окне: $100$.
    - Интервал между измерениями $\Delta t_{sens}$: $0.001$ с (1 мс).
    - **Обоснование (Критерий Найквиста)**: Частота дискретизации составляет $f_s = 1000$ Гц. Согласно критерию Найквиста-Шеннона, для предотвращения эффекта алиасинга (ложного отображения частоты) частота сэмплирования должна быть как минимум в два раза выше максимальной частоты сигнала. 
    - **Оценка частоты сигнала**: Исходя из теории смазки, частота волн $f = v / \lambda$. При типичной скорости потока $v \approx 5$ м/с и длине волны $\lambda \approx 0.1$ м, частота составляет $\approx 50$ Гц. Даже при более экстремальных параметрах частота редко превышает $100\text{--}200$ Гц. Таким образом, выбранный шаг в $1$ мс ($1000$ Гц) обеспечивает многократный запас и гарантирует точное восстановление сигнала без потерь.

## 4. Структура данных (Data Format)
Каждый экземпляр в датасете содержит:

### Входные признаки (Features):
1. **Глобальные константы**: $[\rho, \sigma]$.
2. **Данные датчиков**: Набор пар $\{(x_i, \mathbf{h}_i)\}_{i=1}^k$, где:
    - $x_i$: Координата датчика.
    - $\mathbf{h}_i$: Вектор из 100 значений толщины пленки, полученный с интервалом $\Delta t_{sens}$.

### Целевые значения (Labels):
- $[\mu, h_{final}]$
где $h_{final}$ — средняя толщина пленки по всей высоте стенки в конце периода наблюдения.

## 5. Техническая реализация и контроль качества
Для обеспечения высокого качества обучающей выборки и стабильности модели применяются следующие механизмы:

### 5.1. Фильтрация данных (Quality Control)
Каждый сгенерированный пример проходит проверку на физическую и численную достоверность. Пример удаляется из датасета, если:
- **Численный взрыв**: В массиве значений толщины $h$ обнаружены `NaN` или `Inf`.
- **Аномальная амплитуда**: Максимальное значение толщины в конце симуляции превышает $10 \cdot h_0$, что свидетельствует о потере устойчивости метода.
- **Высокочастотный шум**: Среднеквадратичное отклонение (STD) значений толщины в конце симуляции превышает допустимый порог, указывая на возникновение нефизических осцилляций.

### 5.2. Предобработка данных (Preprocessing)
Ввиду значительной разности масштабов входных признаков (например, $\rho \approx 1000$ и $\sigma \approx 0.05$), перед подачей в модель применяется нормализация:
- **Стандартизация (StandardScaler)**: Приведение признаков к среднему $0$ и стандартному отклонению $1$.
- **Масштабирование (MinMaxScaler)**: Сжатие значений в диапазон $[0, 1]$ или $[-1, 1]$.
Выбор конкретного метода будет определен в ходе экспериментов с разными архитектурами.

### 5.3. Разделение выборки (Data Splitting)
Для корректной оценки обобщающей способности модели данные разделяются в следующей пропорции:
- **Обучающая выборка (Train)**: $80\%$ — используется для настройки весов модели.
- **Валидационная выборка (Val)**: $10\%$ — используется для подбора гиперпараметров и предотвращения переобучения.
- **Тестовая выборка (Test)**: $10\%$ — используется для финальной оценки точности на данных, которые модель никогда не видела.

### 5.4. Формат хранения
- **Формат**: HDF5 (рекомендуется для больших массивов временных рядов) или CSV.
- **Цель**: Создание выборки, охватывающей как режимы с ярко выраженными волнами, так и режимы сглаженного течения.

## 6. Программная реализация генератора
Процесс автоматизирован с помощью скрипта `src/data_gen/generator.py`.

### Ключевые особенности реализации:
- **Параллельные вычисления**: Используется `ProcessPoolExecutor` для распределения нагрузки по всем доступным ядрам процессора (оптимизировано под многопоточные CPU, например i5-14400F).
- **Высокая скорость**: Благодаря использованию JIT-компиляции Numba в физической модели, время генерации одного примера сведено к минимуму.
- **Система шардирования**: Для предотвращения создания избыточно больших файлов данные разбиваются на «шарды» (файлы по 10 000 примеров в формате `.h5`).
- **Механизм возобновления (Resume)**: Скрипт при запуске анализирует существующие шарды в папке `data/` и продолжает генерацию с того места, где она была прервана.
- **Мониторинг**: В консоли отображается прогресс-бар (`tqdm`), показывающий количество сгенерированных примеров в реальном времени.

### Запуск генератора:
Генерация запускается через главный файл проекта:
```bash
python main.py
```
Затем выберите пункт **1 - Generate Dataset**.
