# Спецификация генерации датасета

Данный документ описывает процесс создания синтетического набора данных для обучения и валидации ML-моделей. Датасет генерируется на основе физической модели волн Капицы.

## 1. Параметрическое пространство (Sampling)
Для обеспечения репрезентативности данных и предотвращения смещения выборки (bias), используется два типа распределений при выборе параметров:

| Параметр | Обозначение | Диапазон значений | Распределение | Единицы измерения |
| :--- | :---: | :---: | :---: | :---: |
| **Динамическая вязкость** | $\mu$ | $[0.001, 1.0]$ | Log-uniform | Па·с |
| **Средняя толщина пленки** | $h_0$ | $[0.0002, 0.002]$ | Log-uniform | м |
| **Плотность жидкости** | $\rho$ | $[800, 1200]$ | Uniform | кг/м³ |
| **Поверхностное натяжение** | $\sigma$ | $[0.02, 0.08]$ | Uniform | Н/м |

**Обоснование Log-uniform сэмплирования**: 
Для параметров $\mu$ и $h_0$, значения которых охватывают несколько порядков величины, обычное равномерное распределение привело бы к недопредставленности малых значений. Логарифмическое сэмплирование обеспечивает равную вероятность выбора значения из каждого порядка (например, интервал $[0.001, 0.01]$ будет представлен так же часто, как $[0.1, 1.0]$), что критически важно для обучения нейросети распознавать различные режимы течения.

**Примечание**: Высота стенки $L = 0.5$ м остается константой для всех сценариев.

## 2. Конфигурация датчиков (Sensor Setup)
Для обеспечения универсальности модели, схема расположения датчиков рандомизируется для каждого примера:
- **Количество датчиков ($k$)**: Случайное целое число в диапазоне от $1$ до $10$.
- **Позиции датчиков ($x_i$)**: Случайные точки в интервале $[0.05, 0.45]$ м. Это позволяет избежать краевых эффектов у входа и выхода.
- **Сортировка**: Позиции датчиков всегда сортируются по возрастанию $x_1 < x_2 < \dots < x_k$.

## 3. Временные параметры сбора данных
Для оптимизации времени генерации и обеспечения репрезентативности сигнала установлены следующие параметры:
- **Общее время симуляции**: $0.2 \dots 0.3$ с. Этого достаточно для выхода системы на установившийся режим волнового течения.
- **Окно наблюдения ($T_{window}$)**: Последние $0.1$ с симуляции используются для формирования временного ряда.
- **Дискретизация**: 
    - Количество измерений в окне: $100$.
    - Интервал между измерениями $\Delta t_{sens}$: $0.001$ с (1 мс).
    - **Обоснование (Критерий Найквиста)**: Частота дискретизации составляет $f_s = 1000$ Гц. Согласно критерию Найквиста-Шеннона, для предотвращения эффекта алиасинга (ложного отображения частоты) частота сэмплирования должна быть как минимум в два раза выше максимальной частоты сигнала. 
    - **Оценка частоты сигнала**: Исходя из теории смазки, частота волн $f = v / \lambda$. При типичной скорости потока $v \approx 5$ м/с и длине волны $\lambda \approx 0.1$ м, частота составляет $\approx 50$ Гц. Даже при более экстремальных параметрах частота редко превышает $100\text{--}200$ Гц. Таким образом, выбранный шаг в $1$ мс ($1000$ Гц) обеспечивает многократный запас и гарантирует точное восстановление сигнала без потерь.

## 4. Структура данных (Data Format)
Каждый экземпляр в датасете содержит:

### Входные признаки (Features):
1. **Глобальные константы**: $[\rho, \sigma]$.
2. **Данные датчиков**: Набор пар $\{(x_i, \mathbf{h}_i)\}_{i=1}^k$, где:
    - $x_i$: Координата датчика.
    - $\mathbf{h}_i$: Вектор из 100 значений толщины пленки, полученный с интервалом $\Delta t_{sens}$.

### Целевые значения (Labels):
- $[\mu, h_0]$

## 5. Техническая реализация
- **Формат хранения**: HDF5 (рекомендуется для больших массивов временных рядов) или CSV.
- **Цель**: Создание выборки, охватывающей как режимы с ярко выраженными волнами, так и режимы сглаженного течения (при высоких $\mu$ или малых $h_0$).
