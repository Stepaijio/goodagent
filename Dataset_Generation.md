# Спецификация генерации датасета

Данный документ описывает процесс создания синтетического набора данных для обучения и валидации ML-моделей. Датасет генерируется на основе физической модели волн Капицы.

## 1. Параметрическое пространство (Sampling)
Для каждого примера в датасете случайным образом выбираются физические параметры из заданных диапазонов:
- **Динамическая вязкость ($\mu$)**: Основной целевой параметр. Сэмплируется в диапазоне от низковязких (вода) до высоковязких (масла) жидкостей.
- **Средняя толщина пленки ($h_0$)**: Второй целевой параметр. Варьируется в пределах типичных значений для пленочных аппаратов.
- **Плотность ($\rho$) и Поверхностное натяжение ($\sigma$)**: Варьируются в пределах физически возможных значений для исследуемых жидкостей.

## 2. Конфигурация датчиков (Sensor Setup)
Чтобы модель была универсальной и устойчивой к количеству оборудования, для каждого сценария генерируется уникальная схема расположения датчиков:
- **Количество датчиков ($k$)**: Случайное число в диапазоне от 1 до $N_{max}$ (например, 1–10).
- **Позиции датчиков ($x_i$)**: Случайные точки вдоль высоты стенки $L$, отсортированные по возрастанию.
- **Регистрация данных**: В каждой точке $x_i$ записывается временной ряд толщины $h(x_i, t)$ с фиксированным шагом дискретизации $\Delta t_{sens}$.

## 3. Структура данных (Data Format)
Один экземпляр данных (один сценарий течения) представляет собой:

### Входные признаки (Features):
1. **Константы**: $[\rho, \sigma]$
2. **Данные датчиков**: Набор пар $\{(x_1, \mathbf{h}_1), (x_2, \mathbf{h}_2), \dots, (x_k, \mathbf{h}_k)\}$, где $\mathbf{h}_i$ — вектор значений толщины во времени.

### Целевые значения (Labels):
- $[\mu, h_0]$

## 4. Технические параметры генерации
- **Окно наблюдения ($T_{window}$)**: Фиксированный временной интервал, достаточный для развития неустойчивости и формирования волн.
- **Частота дискретизации ($\Delta t_{sens}$)**: Выбирается исходя из критерия Найквиста, чтобы избежать алиасинга при фиксировании частоты волн Капицы.
- **Формат хранения**: Данные сохраняются в формате HDF5 или CSV для обеспечения быстрого доступа при обучении нейросетей.
