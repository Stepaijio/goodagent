# Архитектуры моделей машинного обучения для определения параметров пленки

Данный документ описывает три стратегии реализации ML-модели для предсказания динамической вязкости $\mu$ и средней толщины пленки $h_{final}$ на основе временных рядов толщины $h(t)$, полученных с датчиков.

## Общие входные данные
Для всех вариантов используются следующие данные:
- **Временные ряды**: $h_i(t)$ для каждого датчика $i \in \{1, \dots, k\}$.
- **Координаты**: Позиции датчиков $x_i$ вдоль стенки.
- **Известные константы**: Плотность жидкости $\rho$ и коэффициент поверхностного натяжения $\sigma$.

**Целевые параметры предсказания**: Динамическая вязкость $\mu$ и средняя толщина пленки в установившемся режиме $h_{final}$.

---

## Вариант 1: Независимый бустинг (Простой)
Этот подход рассматривает каждый датчик как отдельный источник информации и не учитывает их взаимное расположение.

### Логика работы:
1. **Извлечение признаков**: Для каждого временного ряда $h_i(t)$ вручную вычисляются статистические признаки (среднее, дисперсия, амплитуда, доминирующая частота через БПФ).
2. **Предсказание**: Обучается модель градиентного бустинга (например, XGBoost или CatBoost), которая по признакам одного датчика предсказывает $\mu_i$ и $h_{final,i}$.
3. **Агрегация**: Итоговые значения определяются как среднее арифметическое предсказаний по всем доступным датчикам:
   $$\mu_{final} = \frac{1}{k} \sum_{i=1}^k \mu_i, \quad h_{final} = \frac{1}{k} \sum_{i=1}^k h_{final,i}$$

**Плюсы**: Высокая скорость обучения, интерпретируемость.
**Минусы**: Игнорируется пространственная связь между датчиками (сдвиг фазы волн), что является ключевым физическим признаком.

Подробный анализ и результаты реализации доступны в файле `ML_Variant1_Analysis.md`.

## Вариант 2: Конкатенация признаков (Средний)
Этот подход пытается объединить информацию от всех датчиков в один вектор перед финальным предсказанием.

### Логика работы:
1. **Векторизация**: Каждый сигнал $h_i(t)$ пропускается через простой энкодер (например, небольшую 1D-CNN или LSTM), который сжимает ряд в вектор признаков $v_i$.
2. **Склейка (Concatenation)**: Векторы всех датчиков объединяются в один общий вектор $V_{total} = [v_1, v_2, \dots, v_k]$. Если количество датчиков $k$ меняется, используется Zero-padding или Mean-pooling для приведения к фиксированному размеру.
3. **Предиктор**: К вектору $V_{total}$ добавляются константы $\rho$ и $\sigma$. Полученный массив подается в полносвязную нейросеть (MLP).
4. **Выход**: Прямое предсказание $[\mu, h_{final}]$.

**Плюсы**: Модель видит общую картину по всем датчикам сразу.
**Минусы**: Проблема «проклятия размерности» при большом количестве датчиков; фиксированная структура входа затрудняет работу с произвольным $k$.

Подробный анализ и результаты реализации доступны в файле `ML_Variant2_Analysis.md`.

## Вариант 3: Иерархический механизм внимания (SOTA)
Наиболее продвинутый подход, использующий механизмы внимания (Attention) на двух уровнях: временном и пространственном.

### Логика работы:
1. **Temporal Attention (Уровень сигнала)**:
   - Сигнал каждого датчика обрабатывается через Transformer Encoder (или PatchTST).
   - Механизм внимания внутри ряда позволяет выделить наиболее значимые участки волны, игнорируя шум.
   - **Результат**: Высокоинформативный вектор $v_i$ для каждого датчика.
2. **Spatial Attention (Уровень датчиков)**:
   - Все векторы $\{v_1, \dots, v_k\}$ вместе с их координатами $x_i$ подаются в Set Transformer или блок Multi-Head Attention.
   - Модель динамически определяет важность каждого датчика и анализирует взаимосвязи (корреляции, сдвиги) между ними.
   - **Результат**: Глобальный вектор состояния системы $V_{global}$.
3. **Финальный предиктор**:
   - $[V_{global}, \rho, \sigma] \rightarrow \text{MLP} \rightarrow [\mu, h_{final}]$.

**Плюсы**: Полная инвариантность к количеству датчиков, учет физической структуры течения, максимальная точность.
**Минусы**: Высокая вычислительная сложность, требуется больше данных для обучения.

Подробный анализ и результаты реализации доступны в файле `ML_Variant3_Analysis.md`.

## Вариант 4: Обогащенный бустинг (Enriched Boosting)
Этот подход развивает идеи Варианта 1, фокусируясь на глубоком инжиниринге физически значимых признаков вместо усложнения архитектуры нейросети.

### Логика работы:
1. **Обогащение признаков**: Для каждого датчика вычисляется расширенный набор дескрипторов:
   - Частотные характеристики (БПФ: доминирующая частота, энергия).
   - Статистика временного ряда (амплитуда, дисперсия).
   - Динамические признаки (скорость изменения $\Delta h$).
   - Пространственные связи (кросс-корреляция с соседями, временные сдвиги).
2. **Предсказание**: Обучается модель градиентного бустинга (CatBoost/XGBoost) на этом расширенном векторе.
3. **Агрегация**: Итоговое значение — среднее по всем датчикам.

**Плюсы**: Высочайшая точность на текущем объеме данных, физическая интерпретируемость признаков.
**Минусы**: Требует ручного подбора признаков (feature engineering).

Подробный анализ и результаты реализации доступны в файле `ML_Variant4_Analysis.md`.
