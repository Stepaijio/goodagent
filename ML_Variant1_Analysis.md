# Анализ реализации Модели Варианта 1: Независимый бустинг

Данный документ описывает процесс поиска оптимальной предобработки данных для базовой модели (Baseline) и выводы, которые станут фундаментом для всех последующих итераций проекта.

## Архитектура модели
- **Подход**: Независимая обработка датчиков. Каждый датчик в одном примере рассматривается как отдельный обучающий объект.
- **Алгоритм**: Градиентный бустинг на решающих деревьях (`CatBoostRegressor`).
- **Входные данные**: Вектор из 102 признаков (2 глобальные константы $\rho, \sigma$ + 100 точек временного ряда $h(t)$).
- **Целевые переменные**: $\log_{10}(\mu)$ для вязкости и $h_{final}$ для толщины.
- **Агрегация**: Итоговое предсказание для примера получается путем усреднения ответов по всем $k$ датчикам.

## 1. Первый этап: Работа с сырыми данными (Baseline)

На первом этапе мы использовали абсолютные значения толщины пленки $h(t)$ в качестве признаков.

- **Признаки**: $[\rho, \sigma, h_1, \dots, h_{100}]$.
- **Результат по вязкости ($\mu$)**: $R^2 \approx 0.14$.
- **Результат по толщине ($h_{final}$)**: $R^2 \approx 0.99$.

**Анализ проблемы:**
Предсказание толщины оказалось тривиальным, так как модель просто усредняла входной ряд. Предсказание вязкости было крайне слабым ($R^2 = 0.14$).

---

## 2. Второй этап: Внедрение физической нормализации и корректная процедура обучения

Было обнаружено, что предыдущие результаты страдали от некорректной процедуры обучения (утечка данных при нормализации).

**Что было сделано:**
1. **Относительная нормализация**: Вместо сырых значений $h(t)$ подаются отклонения от среднего: 
   $$\text{feature} = \frac{h(t) - h_{mean}}{h_{mean}}$$
2. **Корректная процедура обучения**: `StandardScaler` обучается строго на тренировочной выборке.

---

## 3. Сравнение результатов

После исправления процедур мы получили следующие показатели:

| Параметр | $R^2$ (Сырые данные) | $R^2$ (Нормализованные данные) | Эффект |
| :--- | :---: | :---: | :---: |
| **Log-Viscosity ($\log_{10}\mu$)** | $0.1407$ | **$0.8228$** | $\uparrow$ **Значительный рост** |
| **Final Thickness ($h_{final}$)** | $0.9996$ | $0.8293$ | $\uparrow$ Стабильный результат |

**Вывод по результатам:**
Переход от $R^2 = 0.14$ к $0.82$ подтверждает, что корректная подготовка данных (физическая нормализация + корректная процедура сплита) является критически важным фактором для предсказательной способности модели. Теперь даже простая модель бустинга эффективно извлекает сложные зависимости из формы волны.

---

## 4. Стратегические решения для Вариантов 2 и 3

На основе этих экспериментов приняты следующие решения для разработки сложных нейросетевых моделей:

1. **Обязательная нормализация**: Все последующие архитектуры (CNN, Transformer) должны получать на вход только относительные отклонения $\frac{h - h_{mean}}{h_{mean}}$.
2. **Специализация на вязкости**: Основной фокус в Вариантах 2 и 3 смещен на предсказание вязкости $\mu$, так как предсказание $h_{final}$ является менее сложной задачей.
