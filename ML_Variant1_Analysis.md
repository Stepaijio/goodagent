# Анализ реализации Модели Варианта 1: Независимый бустинг

Данный документ описывает подход, результаты и выводы по первой итерации разработки ML-модели для определения параметров жидкой пленки.

## 1. Архитектура модели
Вариант 1 был выбран в качестве базового уровня (Baseline) для определения минимально возможной точности предсказания без использования сложных нейросетевых структур.

**Основные принципы:**
- **Независимая обработка датчиков**: Каждый датчик в одном примере рассматривался как отдельный независимый обучающий объект. Если в примере было $k$ датчиков, в обучающую выборку добавлялось $k$ строк.
- **Алгоритм**: Использовался градиентный бустинг на решающих деревьях (CatBoost/XGBoost), который эффективно работает с табличными данными.
- **Входные данные**: Вектор из 102 признаков: 100 значений временного ряда толщины $h(t)$ и 2 глобальные константы ($\rho, \sigma$).
- **Целевые переменные**: 
    - Для вязкости: $\log_{10}(\mu)$ (для компенсации логарифмического распределения параметра).
    - Для толщины: $h_{final}$ (среднее значение по всей стенке).
- **Агрегация**: Финальный ответ для примера получался путем усреднения предсказаний всех $k$ датчиков.

## 2. Результаты обучения

По итогам тестирования на отложенной выборке были получены следующие метрики:

| Параметр | MAE (Средняя абс. ошибка) | $R^2$ Score (Коэф. детерминации) |
| :--- | :---: | :---: |
| **Log-Viscosity ($\log_{10}\mu$)** | $0.677440$ | $0.1407$ |
| **Final Thickness ($h_{final}$)** | $0.000005$ | $0.9996$ |

---

## 3. Анализ результатов

### Почему предсказание толщины ($h_{final}$) оказалось почти идеальным?
Высокий $R^2$ для толщины объясняется тем, что $h_{final}$ является средним значением профиля. Даже самый простой алгоритм бустинга легко вычисляет среднее значение по 100 точкам временного ряда. Эта задача оказалась тривиальной, так как не требовала анализа динамики или структуры сигнала.

### Почему предсказание вязкости ($\mu$) оказалось слабым?
Низкий $R^2$ (0.14) указывает на то, что модель почти не смогла уловить зависимость между сырыми данными и вязкостью. Основные причины:
1. **Отсутствие понимания временной структуры**: Бустинг воспринимает 100 точек ряда как 100 независимых колонок. Он не понимает, что точка $t_1$ предшествует точке $t_2$. Таким образом, информация о частоте и фазе волн (которые напрямую зависят от $\mu$) теряется.
2. **Отсутствие пространственного контекста**: Поскольку датчики обрабатываются независимо, модель не видит, как волна перемещается от одного датчика к другому. Скорость этого перемещения — ключевой признак для определения вязкости.
3. **Чувствительность к шуму**: При подаче сырых данных модель склонна переобучаться на случайных флуктуациях в конкретных точках времени, вместо того чтобы извлекать общие физические закономерности.

---

## 4. Выводы
Вариант 1 подтвердил, что:
- Простой регрессионный анализ сырых данных недостаточен для идентификации сложных физических свойств жидкости.
- Для определения вязкости необходимо использовать архитектуры, способные извлекать признаки из временных последовательностей (CNN, RNN) и учитывать взаимодействие между несколькими датчиками (Attention).

Данный результат стал отправной точкой для перехода к более сложным нейросетевым архитектурам (Варианты 2 и 3).
