# Анализ реализации Модели Варианта 2: CNN-MLP

Данный документ описывает процесс разработки, итерации и результаты обучения второй версии модели машинного обучения для определения параметров жидкой пленки.

## 1. Архитектура модели
Вариант 2 представляет собой гибридную архитектуру, сочетающую сверточные слои для извлечения признаков из временных рядов и полносвязную сеть для финального предсказания.

**Основные компоненты:**
- **Shared CNN Encoder**: Каждый временной ряд каждого датчика проходит через одну и ту же сеть сверток (1D-CNN). Это позволяет модели извлекать инвариантные признаки формы волны.
- **Global Average Pooling (GAP)**: Вместо склеивания всех признаков всех датчиков, используется усреднение признаков по всем активным датчикам. Это делает модель инвариантной к количеству установленных датчиков.
- **MLP Predictor**: Финальный вектор (агрегированные признаки + глобальные константы $\rho, \sigma$) подается в многослойный перцептрон для предсказания $\log_{10}(\mu)$ и $h_{final}$.

---

## 2. История итераций и оптимизаций

### Итерация 1: Базовая реализация
**Подход**: Подача сырых данных $h(t)$ в CNN $\rightarrow$ Склейка всех признаков всех датчиков $\rightarrow$ MLP.
**Результаты**:
- **Log-Viscosity**: $R^2 \approx 0.16$ (незначительный рост по сравнению с бустингом).
- **Final Thickness**: $R^2 \approx 0.07$ (очень слабый результат).

**Проблема**: Модель испытывала трудности с предсказанием толщины, так как абсолютные значения $h$ подавляли относительную динамику волн, а огромный размер вектора после склейки приводил к переобучению.

### Итерация 2: Внедрение нормализации и двухпоточного входа
Для исправления ошибок была применена серия улучшений:
1. **Нормализация сигнала**: Введен расчет относительной толщины $h_{norm} = \frac{h - h_{mean}}{h_{mean}}$. Это позволило модели фокусироваться на амплитуде и форме волны, не отвлекаясь на базовый уровень толщины.
2. **Двухканальный вход**: Чтобы вернуть модели способность предсказывать абсолютную толщину $h_{final}$, на вход CNN теперь подаются два канала: $[h_{raw}, h_{norm}]$.
3. **Стабилизация**: Добавлены слои `BatchNorm1d` для ускорения сходимости и предотвращения затухания градиентов.
4. **Агрегация**: Переход от склейки (Concatenation) к усреднению (Global Average Pooling), что сократило размерность входного вектора MLP и повысило обобщающую способность.

## 4. Выводы и анализ

1. **Эффективность CNN**: Переход от простых статистических признаков к сверточным слоям позволил увеличить точность определения вязкости ($\mu$). Модель начала улавливать локальные паттерны сигнала.
2. **Конфликт задач**: Результаты подтвердили, что предсказание $\mu$ и $h_{final}$ требуют разных типов данных (относительных и абсолютных). Двухпоточный вход успешно решил эту проблему, подняв $R^2$ толщины с $0.07$ до $0.40$.
3. **Ограничение архитектуры**: Несмотря на прогресс, $R^2 = 0.25$ для вязкости остается недостаточным. Это связано с тем, что усреднение признаков по датчикам (GAP) уничтожает информацию о **пространственном сдвиге** волн. 

**Заключение**: Вариант 2 доказал преимущебу глубокого обучения над бустингом для данной задачи, но также выявил необходимость в использовании механизмов внимания (Attention) для анализа взаимодействия между датчиками, что станет основой Варианта 3.
