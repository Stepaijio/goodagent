# Анализ реализации Модели Варианта 2: CNN-MLP с конкатенацией признаков

Данный документ описывает результаты применения глубокого обучения (1D-CNN) для определения динамической вязкости $\mu$ и сравнительный анализ с базовым уровнем (Вариант 1).

## 1. Архитектура модели

Для Варианта 2 была реализована архитектура, способная автоматически извлекать признаки из временных рядов каждого датчика.

- **Энкодер (1D-CNN)**: 
    - Три слоя сверток с ReLU.
    - Использование MaxPool1d для снижения размерности.
    - AdaptiveAvgPool1d на выходе для получения фиксированного вектора признаков $v_i$ (размерность 64) для каждого датчика.
- **Агрегация (Fusion)**: 
    - **Mean Pooling**: Усреднение векторов всех активных датчиков: $V_{global} = \frac{1}{k} \sum v_i$.
- **Предиктор (MLP)**:
    - Полносвязная сеть, принимающая вектор $[V_{global}, \rho_{norm}, \sigma_{norm}]$.
    - Выход: одно значение $\log_{10}(\mu)$.

## 3. Выводы

Хотя Вариант 2 (CNN-MLP) после исправления нормализации показал высокую точность ($R^2 \approx 0.80$), он всё еще немного уступает бустингу (Вариант 1).

Причины отставания остаются прежними:
1. **Потеря пространственной информации**: Использование `Mean Pooling` по датчикам игнорирует информацию о взаимосвязях (фазовом сдвиге) между датчиками.
2. **Превосходство бустинга на малых данных**: Модели на основе деревьев лучше справляются с предоставленными 100 000 примеров, тогда как нейросети требуют более сложной настройки для достижения аналогичного качества.

**Итог:** Модель доказала свою работоспособность, но подтвердила необходимость более продвинутых механизмов агрегации пространственных данных, таких как Attention (Вариант 3).
