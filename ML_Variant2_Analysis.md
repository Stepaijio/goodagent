# Анализ реализации Модели Варианта 2: CNN-MLP с конкатенацией признаков

Данный документ описывает результаты применения глубокого обучения (1D-CNN) для определения динамической вязкости $\mu$ и сравнительный анализ с базовым уровнем (Вариант 1).

## 1. Архитектура модели

Для Варианта 2 была реализована архитектура, способная автоматически извлекать признаки из временных рядов каждого датчика.

- **Энкодер (1D-CNN)**: 
    - Три слоя сверток с BatchNorm и ReLU.
    - Использование MaxPool1d для снижения размерности.
    - AdaptiveAvgPool1d на выходе для получения фиксированного вектора признаков $v_i$ (размерность 64) для каждого датчика.
- **Агрегация (Fusion)**: 
    - **Mean Pooling**: Для обеспечения инвариантности к количеству датчиков $k$ использовалось усреднение векторов всех активных датчиков: $V_{global} = \frac{1}{k} \sum v_i$.
- **Предиктор (MLP)**:
    - Полносвязная сеть, принимающая вектор $[V_{global}, \rho_{norm}, \sigma_{norm}]$.
    - Выход: одно значение $\log_{10}(\mu)$.

## 3. Анализ причин низкой эффективности

Несмотря на использование нейросетей, Вариант 2 показал результаты значительно хуже, чем простой бустинг. Основные причины:

1. **Потеря пространственной информации**: Использование `Mean Pooling` по датчикам полностью уничтожает информацию о взаимосвязях между ними. Вязкость $\mu$ напрямую влияет на скорость перемещения волны от одного датчика к другому (сдвиг фазы). Усреднение признаков «стирает» этот ключевой физический признак.
2. **Переобучение (Overfitting)**: Анализ кривых потерь (`loss_curve.png`) показал значительный разрыв между Train Loss ($\sim 0.40$) и Val Loss ($\sim 0.77\text{--}1.07$). Модель склонна запоминать конкретные примеры из обучающей выборки, вместо того чтобы извлекать общие физические закономерности.
3. **Избыточность архитектуры**: Для текущего объема данных 1D-CNN может быть слишком сложной, что в сочетании с простым усреднением признаков приводит к ухудшению обобщающей способности.

## 4. Выводы и переход к Варианту 3

Эксперимент с Вариантом 2 подтвердил следующее:
- **Простое усреднение признаков датчиков недостаточно** для определения вязкости, так как оно игнорирует пространственную структуру течения.
- **Базовая модель с физической нормализацией** является очень сильным конкурентом, и для её превзойти необходим более интеллектуальный механизм объединения данных от датчиков.

**Решение для Варианта 3:**
Для устранения проблем Варианта 2 необходимо заменить `Mean Pooling` на **механизм внимания (Spatial Attention)**. Это позволит модели:
- Динамически определять важность каждого датчика.
- Анализировать корреляции и сдвиги между сигналами разных датчиков, тем самым учитывая скорость распространения волн.
- Сохранить инвариантность к количеству датчиков $k$ без потери важной информации.
