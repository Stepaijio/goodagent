# Структура программной реализации проекта

Данный документ описывает назначение директорий и файлов в проекте по моделированию динамики жидких пленок.

## Директории

- `src/` — Основной исходный код проекта.
    - `physics/` — Реализация физической модели (уравнение эволюции толщины пленки, расчет волн Капицы).
    - `ml/` — Модели машинного обучения: архитектура нейросетей, функции потерь, процесс обучения и валидации.
    - `utils/` — Вспомогательные инструменты: работа с конфигурациями, логгирование, математические утилиты.
    - `data_gen/` — Скрипты для автоматизированной генерации синтетического датасета с помощью физической модели.
- `data/` — Хранилище сгенерированных данных (наборы временных рядов в форматах CSV или HDF5).
- `notebooks/` — Jupyter-ноутбуки для быстрого прототипирования, визуализации течения и анализа данных.
- `tests/` — Unit-тесты для проверки корректности физических расчетов и функций предобработки данных.

## Файлы

- `main.py` — Главная точка входа в программу (выбор между генерацией данных и визуализацией).
- `requirements.txt` — Список необходимых библиотек (numpy, scipy, pandas, torch, numba, h5py, tqdm).
- `.gitignore` — Список файлов и папок, которые не должны попадать в репозиторий.
- `Project_Overview.md` — Общий обзор целей и задач проекта.
- `Physical_Model.md` — Математическая спецификация модели волн Капицы.
- `ML_Architectures.md` — Описание стратегий и архитектур ML-моделей.
- `Dataset_Generation.md` — Спецификация процесса и параметров генерации данных.
- `Diploma_Worklog.md` — Журнал хода разработки и принятых решений.
- `src/data_gen/generator.py` — Скрипт высокопроизводительной параллельной генерации датасета.
- `src/ml/inspect_example.py` — Инструмент для анализа и визуализации отдельных примеров из HDF5-шардов.
