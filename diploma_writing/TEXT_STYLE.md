## СТРОГИЕ ПРАВИЛА СТИЛЯ

### ЗАПРЕЩЕНО (критические ошибки):

1. **Буллет-поинты и нумерованные списки**
   - ❌ Никаких •, -, *, 1., 2. в основном тексте
   - ✅ Только непрерывная проза в форме параграфов

2. **Длинные тире (—)**
   - ❌ Длинное тире для выделения вставных конструкций
   - ✅ Короткие дефисы для составных слов, запятые для логических связей

3. **Деепричастные обороты**
   - ❌ "Используя трансформеры, модели достигли..."
   - ✅ "Внедрение трансформеров обеспечило качественный прорыв..."

4. **Придаточные с "что делало", "что позволяет", "что делает"**
   - ❌ "Данные архитектуры позволяли учитывать контекст, что делало их пригодными..."
   - ❌ "Механизм внимания позволяет модели взвешивать важность, устраняя необходимость..."
   - ✅ "Данные архитектуры учитывали последовательный контекст и применялись для..."
   - ✅ "Механизм внимания взвешивает важность различных частей входа и устраняет необходимость..."

5. **Эмоциональная окраска**
   - ❌ "К сожалению", "удивительно", "очевидно"
   - ✅ Нейтральный аналитический тон

6. **Вода и общие фразы**
   - ❌ "Как известно", "в наше время", "необходимо отметить"
   - ✅ Каждое предложение содержит конкретную информацию

### ОБЯЗАТЕЛЬНО (требования):

1. **Терминологическая точность**
   - При первом упоминании: "полное название на русском (Полное Название На Английском, АББРЕВИАТУРА)"
   - Пример: "большие языковые модели (Large Language Models, LLM)"
   - Далее используется аббревиатура

2. **Инлайн цитирование**
   - Формат: [https://arxiv.org/pdf/XXXX.XXXXX]
   - Несколько источников: [https://source1, https://source2]
   - Цитаты сразу после утверждения

3. **Причинно-следственные связи**
   - Используй союзы и союзные слова: "поскольку", "благодаря", "в результате", "поэтому"
   - Предпочитай сочинительные союзы "и", "а" для связи независимых действий
   - Каждое предложение логически следует из предыдущего

4. **Конкретика**
   - Годы: "в 2017 году", "с 1990-х годов"
   - Цифры: "до одного миллиарда параметров"
   - Имена: "BERT", "Transformer", "архитектура GPT"

5. **Философское обрамление** (для концептуальных разделов)
   - Структура: тезис → антитезис → синтез
   - Пример из ВВЕДЕНИЕ.md: "тезис - LLM обладают мощнейшими когнитивными способностями для обработки информации; антитезис - отсутствие темпоральной протяженности и персонализации делает их бесполезными для задач, требующих накопления опыта; синтезис - создание архитектур, объединяющих генеративную мощь LLM с внешними модулями долговременной памяти"

6. **Структура параграфа**
   - Первое предложение: введение темы
   - Средние предложения: развитие аргумента с фактами
   - Последнее предложение: переход или вывод
   - Минимум 3 предложения, оптимально 5-8

7. **Активный залог**
   - ✅ "Архитектура трансформера имела два основных преимущества"
   - ❌ "Два основных преимущества были у архитектуры трансформера"

## ПРАВИЛА ОФОРМЛЕНИЯ

### Общий стиль текста

- Стиль глав должен быть простым, лаконичным и поясняющим. Избегать «простыней текста».
- Разбивать сплошной текст на лаконичные последовательные блоки с визуальными «зацепками» (таблицы, диаграммы, определения).

### Заголовки

- Количество уровней заголовков — не более трёх.
- Выравнивание по ширине, без отступов, полужирный шрифт, без точки в конце.
- Заголовки первого уровня — с новой страницы, ПРОПИСНЫМИ буквами.
- Нумерация арабскими цифрами с завершающей точкой: «1.», «2.1.», «3.1.2.». Номер отделяется от текста одиночным пробелом.
- «Оглавление», «Введение», «Заключение», «Литература» — заголовки первого уровня без номера.

### Ссылки на литературу

- Ссылки на литературу оформляются в квадратных скобках: [N].

### Рисунки

- Подпись под рисунком по центру, формат: «Рисунок N – Название», без точки в конце.
- После рисунка — пустая строка.
- На каждый рисунок должна быть ссылка в тексте.

### Таблицы

- Подпись НАД таблицей по центру, формат: «Таблица N – Название», без точки в конце.
- После таблицы — пустая строка.
- На каждую таблицу должна быть ссылка в тексте.

### Аббревиатуры

- При первом упоминании: полное название на русском, затем в скобках полное название на английском и аббревиатура.
- Формат: полное название (Полное Название На Английском, АББРЕВИАТУРА).
- Пример: генеративный предобученный трансформер (Generative Pretrained Transformer, GPT).
